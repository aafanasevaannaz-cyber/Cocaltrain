# Правила для ИИ-агентов

Этот проект должен оставаться рабочим React/Vite-приложением, готовым к деплою на Vercel.

## Обязательные правила

1. После каждой задачи обновлять:
   - `REPORTS/LAST_TASK.md`
   - `REPORTS/STATE.md`
2. Нельзя завершать задачу без короткого отчета: что сделано, какие файлы изменены, что проверить дальше.
3. Нельзя удалять Vite и заменять его самодельным dev/build server.
4. `package.json` должен сохранять команды:
   - `npm run dev` → `vite`
   - `npm run build` → `vite build`
   - `npm run preview` → `vite preview`
5. Проект должен оставаться совместимым с Vercel.
6. Не коммитить секреты, токены, ключи API, приватные данные и локальные `.env`.
7. Не ломать публичную структуру песен в `public/songs` без миграции и описания в отчете.
8. Если меняется формат JSON-песни, обновить README и `REPORTS/STATE.md`.

## Перед завершением задачи

Проверить или честно указать, что не удалось проверить:

```bash
npm install
npm run build
```

## Формат отчета

В `REPORTS/LAST_TASK.md` обязательно заполнить:

- дата
- цель задачи
- что сделано
- какие файлы изменены
- что проверено
- что не проверено
- риски
- следующий шаг

В `REPORTS/STATE.md` обязательно поддерживать актуально:

- что работает
- что в процессе
- известные проблемы
- архитектурные решения
- следующий приоритет
